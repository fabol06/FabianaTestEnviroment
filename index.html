What will happen in this step

What your choices will change

Each choice you make will shape how your agent behaves with real users. Use this space to understand what each decision will change.

Interaction Structure

Defines how predictable or adaptive the conversation with users will feel.

What the agent is asking you: how strictly it should follow a predefined interaction flow when guiding users.
What this decision shapes: how consistent, controlled, or adaptive the interaction will be across different users.
What changes depending on what you choose: a more structured setup creates repeatable and predictable interactions, while a more flexible one allows the agent to adapt to user responses and context.
How to decide: choose more structure when consistency, reliability, or compliance matter most, and more flexibility when users benefit from a more natural, adaptive interaction.

Handling incomplete or uncertain information

Defines how cautious or permissive the agent should be when data is incomplete.

What the agent is asking you: how it should behave when information is missing, uncertain, or only partially matches the request.
What this decision shapes: how much the agent prioritizes strict accuracy versus broader but less certain results.
What changes depending on what you choose: a stricter approach excludes uncertain results to protect reliability, while a more flexible one includes partial matches but clearly signals missing or unclear information.
How to decide: choose stricter handling when trust and accuracy are critical, and more flexibility when it‚Äôs better to show possible options rather than risk hiding useful information.

Output structure

Defines how users will receive and interpret the agent‚Äôs responses.

What the agent is asking you: how structured and detailed its responses should be when presenting results or recommendations.
What this decision shapes: how quickly users can scan, understand, and act on the information provided.
What changes depending on what you choose: a compact structure supports fast scanning and quicker decisions, while a more detailed structure provides richer context but requires more attention.
How to decide: choose compact outputs when speed and clarity matter most, and more detailed outputs when context, comparison, or explanation are important.

Agent Design Cheatsheet

Key terms used throughout the workshop, with practical guidance.

Terms Definition Tips
Prompt / Instructions The ‚Äúbrain‚Äù of the agent: clear goals, scope, guardrails, tone. Write like a policy. Be explicit about what to do and what not to do.
System Prompt (Agent Instructions) Persistent rules that shape all responses and actions. Include persona, JTBD, success criteria, and compliance notes.
Grounding / Knowledge The sources an agent can use (files, sites, data). Curate ‚Äújust enough‚Äù high-quality content; keep it current.
Actions / Connectors The capabilities the agent can execute (tasks, records, APIs). Start read-only; add write actions once guardrails are tested.
Temperature Controls creativity vs. determinism. Lower if the agent rambles or invents details.
Max Tokens / Response Length Caps how long a response can be. Short for chat UX; longer for drafts or summaries.
Guardrails & Safety Filters Policy boundaries and role-based constraints. Encode must/never rules and test with edge cases.
Evaluation / Test Scenarios Repeatable prompts with expected outcomes. Build 10‚Äì20 real prompts and iterate.
Observability (Logs / Telemetry) How you review what the agent did and why. Track failures, refusals, hallucinations, latency.
Publishing & Governance Who can see/use the agent; versioning and control. Maintain an Agent Register (owner, data, risk).
Agent Builder vs. Copilot Studio Fast prototyping vs. deeper customization. Prototype in Agent Builder ‚Üí harden in Copilot Studio.
